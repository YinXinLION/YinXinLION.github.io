Java序列化

序列化：将对象的状态信息转换为可存储或传输的形式过程。
反序列化：将字节数组反序列化为对象。
可以帮助我们，将对象持久化到存储设备，通过网络传输对象等问题。
常用的序列化技术有：java默认序列化、XML、JSON、Hessian

Java默认序列化

Java序列化通过对象输出流java.io.ObjectOutputStream与对象输入流java.io.ObjectInputStream实现，被序列化的类要实现Serializable接口。
transient关键字可以在默认序列化时忽略修饰的字段。

XML序列化

XML序列化，可读性好，但序列化后码流大，效率不高。它具有语言无关性，可用于异构系统之间的数据交换。

Hessian序列化

Hessian是一个支持跨语言传输的二进制序列化协议，支持不同的语言。

  • ByteArrayOutputStream/ByteArrayInputStream可以将字符串或者字节数组转换成输入/输出流。
  • FileOutputStream/FileInputStream专门用于文件IO操作，更适用于图片等二进制文件操作。
  • BufferedOutputStream/BufferedInputStream使用装饰器模式。增加字节流缓存能力。使字节数组能批量读取写入。
  • DataOutputStream/DataInputStream对Java原生类型直接写入再按写入顺序直接读取。常用语网络数据传输过程读取和写入。
  • PrintStream用语日志输出组件的实现，无需抛出IO异常。
  • ObjectOutputStream/ObjectInputStream用语Java对象的序列化与反序列化或网络数据的写入/写出。
